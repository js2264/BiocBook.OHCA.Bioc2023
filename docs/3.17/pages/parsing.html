HiCool vs. hicstuff ðŸ‘‡

HiCool mostly operates as a wrapper to the lightweight python-based hicstuff library. What HiCool really does is running hicstuff pipeline. You can replicate this outside of R console, directly in the terminal, by running:

sh
hicstuff pipeline \
   <hicstuff-options> \
   --genome <genome.fa> \
   <input.R1.fq.gz> \
   <input.R2.fq.gz>

A call to HiCool() triggers hicstuff pipeline to process fastq files into .pairs and .(m)cool files, and returns a CoolFile object in R, along with HTML reports.

library(HiContactsData)

##  see ?HiContactsData and browseVignettes('HiContactsData') for documentation

    r1, r2,

2.1.2 HiCool outputs

You can check the generated output files placed in the HiCool/ directory.

hcf
##  CoolFile object
##  .mcool file: ./HiCool//matrices/2bb1b9087d6_7833^mapped-R64-1-1^WL4DIE.mcool 
##  resolution: 1000 
##  pairs file: ./HiCool//pairs/2bb1b9087d6_7833^mapped-R64-1-1^WL4DIE.pairs 
##  metadata(3): log args stats
  • The *.pairs and *.mcool files are the pairs and contact matrix files, respectively. These are the output files the end-user is generally looking for.
  • The *.html file is a report summarizing pairs numbers, filtering, etcâ€¦
  • The *.log file contains all output and error messages, as well as the full list of commands that have been executed to pre-process the input dataset.
  • The *.pdf graphic files provide a visual representation of the distribution of informative/non-informative pairs.
